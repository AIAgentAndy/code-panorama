# 两周 AI 编程实战教程（10 个工作日）：从 0 到 1 做出代码全景分析器

> 教学目标：带学员在 2 周内（周末休息）完成一个“代码仓库 -> 调用链分析 -> 可视化图谱”的同类产品，重点掌握 AI 编程协作与提示词工程。

## 教学原则

- 不抄现有项目代码，按 MVP -> 迭代增强推进。
- 每天必须有可运行产出。
- 每天都做提示词迭代与复盘。

---

## 第 1 周（5 天）：打通 MVP

### Day 1：项目初始化 + 第一版可运行页面（当天正式编码）

- 目标：第一天就交付可运行代码。
- 产出：Next.js 项目可启动；首页完成仓库输入、开始按钮、日志区；前后端健康检查接口可调用。
- 任务：
  - 用 AI 辅助初始化工程并跑通本地启动。
  - 编写首页基础交互和状态（idle/loading/error）。
  - 新增 `/api/health`（或等价）用于前后端联调验证。
  - 在编码前先用 15 分钟完成“最小任务拆解”（不单独占一天）。
- 提示词训练：让 AI 同时输出“页面骨架 + 类型定义 + 最小接口样例”。

### Day 2：脚手架与基础页面

- 目标：接入真实数据源前的接口骨架搭建。
- 产出：分析请求链路打通（前端提交 -> 后端响应 -> 前端展示）。
- 任务：定义 TypeScript 数据结构、封装 API 调用层、完善日志展示区域。
- 提示词训练：接口契约生成与前后端类型对齐。

### Day 3：GitHub 数据源接入

- 目标：拿到仓库文件树。
- 产出：`/api/github/validate`、`/api/github/tree`。
- 任务：URL 解析、仓库校验、文件树拉取与过滤。
- 提示词训练：异常处理（401/403/404/限流）与重试策略。

### Day 4：文件内容获取 + LLM JSON 网关

- 目标：打通“代码内容 -> AI 结构化输出”。
- 产出：`/api/github/content` + `/api/llm/json`。
- 任务：批量拉取文件内容、大小限制、JSON-only 输出、解析容错、调用日志。
- 提示词训练：System Prompt 固定“仅返回 JSON”。

### Day 5：入口识别 + 单函数分析

- 目标：让 AI 能识别入口并分析函数子调用。
- 产出：入口识别器 + 函数分析器。
- 任务：
  - 入口识别输出：`language`、`entryFile`、`entryFunctionCandidates`。
  - 函数分析输出：`functionName`、`description`、`calls[]`、`shouldDrill`。
- 提示词训练：A/B 两版提示词稳定性对比。

---

## 第 2 周（5 天）：完成核心能力与交付

### Day 6：递归下钻引擎

- 目标：实现调用链自动扩展。
- 产出：调用链生成器 v1（支持深度与子节点数限制）。
- 任务：任务队列、节点状态（queued/analyzing/done/failed）、失败回退。
- 提示词训练：引导模型优先输出关键业务调用。

### Day 7：函数定位多阶段兜底

- 目标：解决“函数名有了但文件未知”。
- 产出：定位策略 v1（猜测文件 -> AI Top3 -> 搜索回退 -> 系统函数停止）。
- 任务：实现多阶段定位与系统/库函数识别。
- 提示词训练：系统函数识别 JSON 规范。

### Day 8：图谱可视化 + 手动下钻

- 目标：把调用链可视化并可交互。
- 产出：Graph 页面（节点/边/详情/手动继续下钻）。
- 任务：接 React Flow、节点信息展示、点击行为与详情联动。
- 提示词训练：节点描述压缩与统一风格生成。

### Day 9：模块划分 + 可观测性

- 目标：从调用链升级到架构视图，并具备调试能力。
- 产出：模块划分结果 + AI 调用统计面板。
- 任务：
  - 模块划分输出：模块列表、节点归属、覆盖率。
  - 记录调用耗时、token、成功率、错误分类。
- 提示词训练：模块一致性约束（命名、数量、全覆盖）。

### Day 10：优化、答辩与沉淀

- 目标：形成可演示可复用成果。
- 产出：可运行 Demo + 讲义 + 提示词模板库 + FAQ。
- 任务：降本优化（上下文裁剪、缓存命中、失败熔断），10 分钟结营 Demo。
- 提示词训练：让 AI 先给优化建议，再人工筛选落地。

## 每日教学节奏（建议）

- 20 分钟：目标与复盘
- 40 分钟：你现场 AI 结对编程演示
- 60 分钟：学员分组实操
- 20 分钟：代码评审（重点：提示词质量、失败处理）
- 10 分钟：日报提交（产出 + 问题 + 明日计划）

## 评分标准（100 分）

- 需求拆解：20 分
- 提示词工程：30 分
- 工程落地：30 分
- 调试复盘：20 分

## 学员必须遵守的 6 条规则

- 先写验收标准，再写提示词。
- AI 产出必须人工审查后再合并。
- 所有关键 AI 调用都用 JSON schema 约束。
- 先看日志再改提示词或代码。
- 每天至少沉淀 1 个失败案例与修复记录。
- 所有实现都要能解释“为什么这么做”。
